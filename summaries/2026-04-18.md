# llama.cpp daily summary — 2026-04-18

Seven commits landed on upstream/master in the past 24 hours, spanning GPU backends, build infrastructure, and model support. The most notable change is a **breaking rename** of `libcommon` → `libllama-common`, which will require downstream projects to update their link targets.

## CUDA/GPU

- **CUDA graph LRU eviction**: Replace unbounded map with LRU-based eviction for per-node CUDA graphs, fixing memory leaks that occurred when `ggml_cgraph` node pointers changed across runs (#21611, `b94050e`)

## WebGPU

- **FlashAttention refactor + warning fixes**: Remove C++20 initializers and unsafe casts, refactor FlashAttention encoding to match the style of other ops; CI on NVIDIA self-hosted node added but not yet fully enabled pending precision investigation (#21052, `45cac7c`)

## OpenCL

- **Adreno q8\_0 dispatch refactor**: Clean up host-side dispatch for `set_tensor` and `mul_mat` on Adreno, aligning with patterns used by newer quantizations like q4\_1 (#21938, `5e6c0e1`)

## Build / **Breaking**

- **`libcommon` → `libllama-common`** ⚠️: Rename the common library and split out `libllama-common-base` (formerly `libbuild_info`); enable shared-library builds; install alongside `libllama`. Downstream users linking `libcommon` must update (#21936, `6990e2f`)

## CI

- **Android arm64 build & release**: Add CI jobs to build and release Android arm64 binaries; also fixes server `ignore_eos` flag handling (#21647, `a279d0f`)

## Model
