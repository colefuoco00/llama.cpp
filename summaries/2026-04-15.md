# llama.cpp daily summary — 2026-04-15

A busy day across GPU backends, with significant work on Vulkan (NVFP4 support and shader precision improvements), Metal fixes, and Hexagon HMX performance. The server gains a new OpenAI-compatible audio transcription endpoint, and a notable sampler bug fix addresses a ~30% throughput regression on Vulkan.

## API / Server

- **OAI `/v1/audio/transcriptions` endpoint** — server now supports the OpenAI audio transcription API with streaming support; tested with gemma-4 (#21863, `e489a5c`)
- **`mtmd_image_tokens_get_decoder_pos()` API** — new multimodal helper returning non-linear t,x,y decoder positions per token; deprecates `get_nx/ny()` (#21851, `707c0b7`)

## Vulkan / GPU

- **NVFP4 support** — adds `get_rows`, dequant, and `mul_mat(_id)` for NVFP4 type via fp16/fp32 path (#21455, `6a6780a`)
- **Programmatic RoundingModeRTE** — shaders now inherit `RoundingModeRTE` from device capabilities at runtime instead of embedding it per-shader; requires SPIRV-Headers (#21572, `1f30ac0`)

## Metal

- **XIELU unary op** — Metal backend gains support for the XIELU activation function (#20802, `aa0f189`)
- **FA support logic fix** — corrects quantization type filtering for the Flash Attention operator (#21898, `c0de6ed`)

## Hexagon

- **HMX mat_mul async + cost-based tuning** — replaces synchronous HMX calls with an hmx-queue, adds cost-aware VTCM chunk search; benchmarks show ~12% pp512 improvement on Elite Gen 5 (#21554, `5d14e5d`)

## ARM / CPU

- **NEON nvfp4 dot product fix** — corrects ARM NEON nvfp4 dot product on targets without dotprod extension (#21559, `2e05f06`)

## WebGPU

- **f32 accumulation in register-tiling matmul** — fixes NaN on older Qwen models and Chrome-specific quant issue; tunes iOS batch size (#21644, `5a23695`)

## Fixes / Performance

- **Reasoning budget sampler skip** — sampler is no longer created when no budget is configured, restoring backend (GPU-side) sampling and fixing ~30% t/s regression on Vulkan (#21870, `56666fa`)
