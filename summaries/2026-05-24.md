# llama.cpp daily summary — 2026-05-24

A light day with 5 commits landed on upstream master, spanning build system improvements, a model update, a backend correctness fix, and two dependency/toolchain patches. The standout change is a significant CMake/UI build refactor that improves incremental build speed dramatically.

## Build / DevOps

- **cmake/ui: refactor the build** — Rewrites the server UI build into a single `ui-assets.cmake` script. Replaces the old `xxd.cmake` approach with a small C++ embed helper, cutting `bundle.js` processing from ~6 s to ~80 ms. Adds incremental builds (skips npm if source timestamps are unchanged), multi-build-tree Vite support, and a prebuilt UI tarball attached to releases for package maintainers. #23352 · `b22ff4b`

## Vulkan

- **Fix Windows `find_package` of SPIRV-Headers** — Corrects a CMake `find_package` call for SPIRV-Headers in the Vulkan backend that was broken on Windows (and in some non-Windows setups). #23215 · `95405ac`

## Models

- **Add NVFP4 MTP scale tensors** — Wires in the missing MTP (Multi-Token Prediction) scale tensors for NVFP4 quantization in the Qwen3.5 and Qwen3.5-MoE model families. #23563 · `b0df4c0`

## ggml / Backend

- **Check correct iface method before fallback 2D get** — Bug fix in `ggml-backend.cpp`: the wrong interface method was being checked before falling back to the 2D buffer `get` path, which could silently select the wrong code path. #23514 · `a497476`

## Dependencies

- **Bump PyTorch to 2.11.0** — Pins `torch==2.11.0` and `torchvision==0.26.0` in `convert_hf_to_gguf` and `mtmd` requirements, fixing a pip install regression caused by the overly strict `~=2.6.0` version specifier on platforms where 2.6.x wheels are unavailable. #23503 · `c0c7e14`
