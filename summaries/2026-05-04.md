# llama.cpp daily summary — 2026-05-04

A quiet Sunday with 3 bugfix commits landing to upstream/master, all targeting the model conversion pipeline and inference correctness. No new features or breaking changes.

## Fixes

- **NVFP4 RoPE permutation bug (critical)** — `convert_hf_to_gguf.py` was missing the Q/K axis-0 row permutation in the NVFP4 repack path, causing scrambled attention heads and gibberish output (perplexity 4419 vs. 44.0 baseline). Fix applies `LlamaModel.permute` to both the nibble-packed weight and per-block scale tensors; all Llama-arch subclasses (Mistral, Granite, Llama4, etc.) inherit the fix automatically. PR #22611 · `db44417`

- **Mistral YaRN `apply_scale` ignored during conversion** — Models with `apply_scale=True` in `params.json` had the flag silently dropped in the Mistral-format conversion path, degrading mid-to-long context performance. Fix threads the value through `convert_hf_to_gguf.py` and `llama-model.cpp`. PR #22612 · `048a490`

- **Generation prompt incorrectly computed for MiniMax M2** — `calculate_diff_split()` in `common/chat.cpp` was returning a wrong generation-prompt prefix for MiniMax M2 models (e.g. `[e~[\n]~b]ai\n<think>\n` instead of `]~b]ai\n<think>\n`), triggering grammar errors when `grammar_lazy = false`. Replaced with a longest-common-prefix approach that is tag-boundary–agnostic. PR #22657 · `e48034d`
