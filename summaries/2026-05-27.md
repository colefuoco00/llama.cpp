# llama.cpp daily summary — 2026-05-27

Today was a busy day with 19 commits hitting upstream/master. The focus was heavily on CI infrastructure reorganization (7 commits by Georgi Gerganov) alongside meaningful backend improvements across Vulkan, WebGPU, Hexagon, CUDA, and SYCL. A new model architecture (talkie-1930-13b) and Gemma4 conversion support round out the day.

## CI/Build

- Disabled SYCL and CANN builds/releases to reduce CI load; renamed caches accordingly (#23705, `3dc7684`)
- Moved macOS jobs to a dedicated `build-apple.yml` workflow (#23721, `5190c2e`)
- Migrated sanitizer and additional CPU jobs to self-hosted runners (#23713, #23715, `ef41a69`, `678d43d`)
- Added `[no release]` commit keyword to skip release workflow on master; moved the check into a dedicated `check_release` job (#23728, #23734, `35a74c8`, `08bc21b`)
- Removed Vulkan SDK dependency from WebGPU CI job and added a WASM test (#23718, `3a3ed15`)
- Stopped allocating ccache for 3rd-party hosted runners (#23730, `0d18aaa`)

## Vulkan

- Significant conv2d optimization: new tile shape (`CONV_SHAPE_64x128`), skip bounds checks when shapes align, WG_SIZE tuning, and added **coopmat1** (cooperative matrix) support for a fast path (#22620, `7799d31`)

## WebGPU

- Added MMVQ (matrix-vector quantized) path for Q4_0, Q8_0, Q2\_K, and Q4\_K quant types; removed the legacy `mul-mat` pipeline — a notable cleanup/perf improvement (#23594, `1506d39`)
- Fixed batch compute passes to be per-thread; submit individual passes when GPU profiling is off (#23457, `54121f7`)

## Hexagon (Qualcomm HTP)

- Added CONCAT op support with an optimized `concat_2d_transposed` path tuned for Qwen3.5 workloads; also optimized CPY, CONT, GET_ROWS (DMA for large rows), and ROPE (HVX vectorized sin/cos) (#23648, `ef66bfa`)

## CUDA/GPU

- Fixed missing PDL sync for FWHT kernel and improved fallback handling (#23690, `192d8ae`)

## SYCL
