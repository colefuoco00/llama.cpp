# llama.cpp daily summary — 2026-05-22

Active day with 19 commits touching GPU backends (CUDA, Vulkan, Metal, WebGPU, Hexagon), server fixes, vocabulary support, build system improvements, and test refactoring. Two notable server-side memory fixes land, and a new tokenizer family is introduced.

## CUDA/GPU

- **CUDA**: Fix PDL CC check for JIT compilation — corrects a condition guard in `common.cuh` that was improperly evaluating compute capability during JIT builds. (#23471, `4f0e43d`)
- **Vulkan**: Fuse snake activation (`mul, sin, sqr, mul, add`) into a single elementwise shader (`snake.comp`) for audio decoders (BigVGAN, Vocos). F32/F16/BF16 pipelines added; tests compare CPU naive vs Vulkan fused. (#22855, `47c0eda`)
- **WebGPU / Flash Attention**: Replace hardcoded `f32` types with `kv_type`/`q_type` in the WebGPU flash-attention WGSL shader, fixing type mismatches for non-F32 KV caches. (#23372, `5306f4b`)
- **WebGPU**: Update WebGPU support docs and link to a new blog/demo. (#23483, `ee7c305`)
- **Hexagon**: Fix ssm-conv for large prompts — removes unnecessary gathers, better VTCM handling, relaxes gating requirements. (#23307, `0be8468`)

## Metal

- **Metal**: Optimize concat kernel with row batching for narrow tensors (`ne0 < 256`), improving GPU occupancy. Also fixes `GGML_OP_SET` kernel thread dispatch. (#23411, `a1a69f7`)

## API / Server

- **Server**: Free draft/MTP resources (`spec`, `ctx_dft`, `model_dft`) on sleep to fix a VRAM leak — on each sleep/resume cycle resources were reallocated without freeing the old ones, eventually crashing with OOM. (#23461, `52fb93a`)
- **Server**: Expose `n_prompt_tokens`, `n_prompt_tokens_processed`, and `n_prompt_tokens_cache` in the `/slots` endpoint response. (#23454, `b65bb4b`)
- **Server**: Re-inject subcommand when router spawns children under the unified binary to prevent misrouting. (#23442, `c902171`)

## Models / Vocab

- **Vocab**: Add `HybridDNATokenizer` (Carbon-3B) as a new first-class vocab type `LLAMA_VOCAB_TYPE_HYBRIDDNA`. DNA text inside `<dna>…</dna>` tags is chunked into 6-mers; bases outside ACGT map to `<oov>`. Conversion support added in `conversion/base.py` and `conversion/llama.py`. (#23410, `7ea23dd`)
- **MTP**: Skip logit computation during follow-up draft-model decode using `inp_out_ids`, avoiding unnecessary work. (#23433, `12e5d99`)

## Build / Tests

- **CMake**: Remove explicit `STATIC` from impl libraries so `BUILD_SHARED_LIBS` controls linkage; enable `LLAMA_BUILD_APP` by default; add `WINDOWS_EXPORT_ALL_SYMBOLS` for proper DLL export. (#23462, `bb28c1f`)
- **App**: Add `batched-bench`, `fit-params`, `quantize`, and `perplexity` subcommands to the unified `llama` app binary. (#23459, `1d7ab2b`)
- **App**: Print version string on startup. (#23426, `ce02093`)
- **Tests**: Move `save-load-state` from `examples/` to `tests/` as a proper model test. (#23336, `40d5358`)
