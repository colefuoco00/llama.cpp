# llama.cpp daily summary — 2026-04-20

A busy day with 11 commits to upstream/master, spanning GPU backend performance, a major new server feature, a breaking C-API change, and build fixes. The headline addition is checkpoint-based speculative decoding in the server; maintainers also landed a notable CUDA/HIP refactor and a performance win in the ggml meta backend.

## CUDA / GPU

- **CUDA: refactor MMA data loading for AMD** — Significant simplification of `mma.cuh` and `mmq.cuh` for AMD targets (CDNA/RDNA3), net -283 lines. Fixes occupancy and compile issues on AMD. (#22051, `4eac5b4`)
- **ggml: cache subgraph splits in meta backend** — Skips per-call subgraph construction when the same `ggml_cgraph` is reused consecutively; assigns UIDs so CUDA's fast-path also hits. Yields up to **1.16× token-gen speedup** on dual RTX 5090. (#22041, `bcdcc10`)
- **HIP: remove unnecessary `NCCL_CHECK`** — One-line cleanup left over from a prior RCCL refactor. (#21914, `471540a`)

## API / Server

- **server: speculative checkpointing** — Adds checkpoint-based speculative decoding to the server (draft model + ngram). Large refactor of `server-context.cpp` and `speculative.cpp`; also enables mtmd speculative decoding. Notable new feature. (#19493, `455d8e4`)

## RPC

- **rpc: refactor RPC transport** — Moves all socket/transport code into `transport.cpp`/`transport.h`, hiding implementation details behind a `socket_t` interface. Reduces `ggml-rpc.cpp` by ~750 lines. (#21998, `91fef95`)

## Multimodal (mtmd) — **Breaking Change**

- **mtmd: add `pos_0` to `mtmd_image_tokens_get_decoder_pos`** — C API change giving models full control over RoPE position dimensions. Callers must update. (#22082, `1912407`)

## Build / CI

- **cmake: revert CMP0194 policy** — Removes a CMake 4.1 policy that broke MSVC builds on Windows runners by causing MinGW to be preferred for ASM. (#21934, `09b4efa`)
- **ci: install `spirv-headers` for `vulkan-cross`** — One-line CI fix for Vulkan cross-compile job. (#22109, `037bfe3`)

## Convert / Tools

- **convert: support sentence-transformer 5.4 config files** — Updates `convert_hf_to_gguf.py` for new pooling and EmbeddingGemma config layouts. (#22087, `8685e7b`)

## Fixes / Autoparser

- **autoparser: allow whitespace after tool call** — Fixes constrained generation for models (e.g. Nemotron Nano 3) that emit a trailing newline after `<tool_call>`. (#22073, `d5b780a`)

## Vendor
