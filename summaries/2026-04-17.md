# llama.cpp daily summary — 2026-04-17

Active day with 18 commits, dominated by large-scale model-layer refactoring that touches nearly every architecture, alongside new hardware backend optimizations and important bug fixes for SYCL and OpenCL. No breaking API changes were introduced, though the model build graph internals were significantly consolidated.

## Model / Architecture

- **QKV refactor across 88 files**: Extracted common `build_qkv` and `create_tensor_qkv` helpers into `llama-graph`, removing ~1400 lines of duplicated code from every model backend. (#21245, `9db77a0`)
- **Single `llm_build` per arch**: Collapsed per-architecture build functions; merges `llama-iswa` → `llama4` and `t5-dec`/`t5-enc` → `t5`. (#21970, `4fbdabd`)
- **NVFP4 tensor support for Gemma4**: Extends `build_attn` with `wo_s` scale and wires NVFP4 quantization through all ~105 model files. (#21971, `f772f6e`)
- **NemotronH converter fix**: Corrects HF config and tokenizer parsing using `trust_remote_code`. (#21664, `03b3d07`)

## Metal

- **ROLL op implemented**: New Metal kernel for the ROLL operation; updates op documentation and Metal CSV capability matrix. (#21946, `ae2d348`)

## OpenCL / Adreno

- **q5\_K GEMM+GEMV kernels for Adreno**: Adds ~900 lines of new OpenCL kernels targeting Qualcomm Adreno mobile GPUs. (#21595, `e45dbde`)

## SYCL

- **Q8\_0 reorder bug fixes** (notable fix): Corrects garbage output on the 2nd prompt and a crash when VRAM is full. Adds reorder-aware DMMV dequantizers for Q8\_0, Q4\_K, Q6\_K; introduces RAII temp-buffer class and a `GGML_SYCL_HOST_MEM_FALLBACK` CMake option (requires Linux ≥6.8). (#21638, `b1be68e`)

## RISC-V

- **128-bit RVV quantization dot products**: Adds 128-bit vector implementations for i-quants, ternary quants, iq2\_xs, iq3\_s, etc. (#20633, `1e796eb`)
- **SIMD GEMM kernel for RVV**: New `simd_gemm` kernel using RISC-V vector extensions. (#20627, `5637536`)

## Hexagon / HTP

- **HMX matmul optimization**: Refactors tile calculations for row/column indices upfront and adds size-limit check for `HAP_mmap`. (#21071, `85dde8d`)

## WebGPU

- **Compute pass batching**: Batches compute passes when not profiling; removes per-pass profiling overhead; consolidates to a single GPU query set. (#21873, `82677a6`)

## ggml core

- **Graph versioning (`graph_reused`)**: Adds a `uid` field to `ggml_cgraph` incremented atomically per split, enabling backends to detect graph reuse without a reuse flag. (#21764, `3f7c29d`)

## Server / API

- **Media marker thread-safety**: Makes `get_media_marker()` thread-safe via C++11 magic statics; adds `LLAMA_MEDIA_MARKER` env-var pin for tests. (#21980, `4adac43`; CLI wired in `30dce2c`)

## Build / CI / DevOps
