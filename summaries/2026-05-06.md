# llama.cpp daily summary — 2026-05-06

A productive day with 12 commits to upstream/master, highlighted by a ggml version bump to 0.11.0 and meaningful improvements across Qualcomm Hexagon/HMX, OpenCL Adreno, RPC, Metal, and the server tooling. A notable new feature adds fast Walsh-Hadamard transform for KV cache rotation, and a new API flag allows KV-cache state to be kept on-device to reduce D2H copy overhead.

## CUDA / GPU

- No CUDA-specific commits landed today.

## OpenCL / Adreno

- **OpenCL: refactor Adreno q4\_0 GEMM/GEMV dispatch** — large cleanup of `ggml-opencl.cpp` (~600 lines removed), consistent kernel naming for q4\_0 and q8\_0 paths. (#22335, `ff806a1`)

## Hexagon / HMX

- **Process M-tail rows on HMX instead of HVX** — when activation row count M is not a multiple of 32, the tail previously fell back to HVX; now zero-padded and dispatched entirely through HMX, improving prefill throughput on Snapdragon 8 Gen 5 for non-power-of-32 token counts. (#22724, `bbeb89d`)

## Metal / Apple

- **Add `cpy_tensor` backend API to Metal** — required to support new on-device KV-cache state storage. (part of `d6e7b03`)

## Core / ggml

- **Bump ggml version to 0.11.0** (`c91faf9`)
- **Implement fast Walsh-Hadamard transform for KV rotation** — new `GGML_OP_RWKV_WKV7` variant using FWHT, CPU backend only for now. (#22631, `a817a22`)
- **Handle non-contiguous Q/K/V in `mul_mat_aux`** — fixes a graph correctness edge case with non-contiguous attention buffers. (#22630, `fa59546`)

## RPC

- **Use graph UID instead of graph cache** — simplifies RPC caching logic by replacing the heavyweight cache map with a single last-seen UID comparison, cutting ~25 lines. (#22701, `d5003b6`)

## API / server

- **Add option to save KV-cache state in device buffers** — new `LLAMA_STATE_SEQ_FLAGS_ON_DEVICE` flag in `llama_state_seq_*_ext` API; avoids D2H copies for speculative decoding checkpoints. (#22679, `d6e7b03`)
- **Validate `--tools` CLI arguments at startup** — unknown tool names now cause an early error listing valid options instead of being silently ignored. (#22538, `2bacb1e`)

## Build / common

- **Only load backends when required** — `llama_backend_init()` now calls `ggml_backend_load_all()` directly, fixing spurious backend loading in tools that don't use `common_params`. (#22290, `bf76ac7`)
- **Fix missing-noreturn warnings for clang 21** — adds `[[noreturn]]` attributes to lambdas in `common/arg.cpp`. (#22702, `2635ac7`)

## Vendor / misc

- **Update cpp-httplib to 0.43.3** — fixes two OSS-Fuzz bugs (Content-Length over-allocation, stack overflow in `str2tag`). (#22686, `a09a00e`)
- **Sync ggml submodule** (`70a8309`)
