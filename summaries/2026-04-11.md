# llama.cpp daily summary — 2026-04-11

A busy day with 15 commits spanning hardware backends, multimodal support, WebGPU improvements, and several model/server fixes. The most significant changes land in the Hexagon/Snapdragon backend (a major rework) and new audio-language model support in the multimodal framework.

## Multimodal / mtmd

- **MERaLiON-2 audio-language model support** — adds A*STAR's MERaLiON-2 (3B and 10B) with a Whisper large-v2 audio encoder, gated MLP adaptor, and Gemma2 decoder; supports transcription in EN/ZH/MS/TA, translation, and spoken QA (`PROJECTOR_TYPE_MERALION`) (#21756, `073bb2c`)

## Hexagon / Snapdragon

- **Major Op queuing, buffer and cache management rework** — introduces batched op dispatch via a single dspqueue message, pinned mmappings, shared-buffer packing, fine-grain L2 cache management, and removes the `GGML_HEXAGON_EXPERIMENTAL` env var in favor of a flexible `GGML_HEXAGON_OPFILTER` regex (~1700 line net change) (#21705, `9aa2807`)
- **Linux/Snapdragon support** — adds Debian/EX2 build config, vectorize flags, installation scripts, and updated docs for running on Linux-on-Snapdragon devices (#21707, `81069a8`)

## WebGPU

- **Quantization precision and backend lifecycle fixes** — corrects F16 NaN canonicalization in packed-integer dequant, fixes `EXP`/`EXPM1`/`sqrt` precision for F16, keeps one Dawn/WebGPU instance alive for the full static backend lifetime, and refactors dequant shader helpers (#21521, `e4fed9d`)
- **Intel GPU non-square subgroup matrix support** — flash attention now handles non-square subgroup matrix configurations (#21669, `bfd1f45`)

## OpenCL

- **Q5_K quantization support** — adds matrix-vector and matrix-matrix kernels for Q5_K in the OpenCL backend (#21593, `af1127d`)

## CUDA

- **Graph equality fix** — stores `node->src` shape/stride (`ne`/`nb`) when checking CUDA graph equality, preventing incorrect graph reuse (#21736, `a29e4c0`)

## Models

- **Gemma 4**: updated chat template to align with the latest official Jinja template (#21704, `3fc6506`); made shared-KV tail `attn_k` tensors optional on load to improve compatibility (#21739, `e62fa13`)
- **Qwen 3 Next**: fix tensor parallelism data split (#21732, `865ff06`)

## API / Server / Common

- **Download progress callback** — new callback interface in `common/download` so callers can track and display download progress (#21735, `05b3caa`)
- **Server `--alias` ignored with `--models-preset`** — prevents a confusing conflict error when both flags are set (#21380, `5dd1025`)
- **Structured output fix** — broken JSON schema output when using `$ref` references is now resolved (#21699, `b136b62`)

## Fixes / Misc

- `ggml`: fix missing `GGML_TYPE_Q1_0` handling in a few CPU ops (#21716, `2b2cd57`)
