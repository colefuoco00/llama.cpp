A note on activation-weighted quantization

The Importance Matrix

A small file, derived from a few hundred thousand tokens, that tells a quantizer where to spend its bits — and where it can afford not to.

i. The Allocation

Quantization is a budget problem. A seven-billion-parameter model has seven billion places where precision can be spent; a Q4_K_M file gives the encoder roughly four bits per parameter to spend. The encoder must decide, weight by weight, how to round. A uniform scheme treats every parameter as equally consequential — but a forward pass through a transformer does not. Some channels carry signal that dominates the next layer's pre-activation; others carry whispers. Squashing a loud channel onto the same coarse grid as a quiet one wastes resolution where it would have done the most good.

The importance matrix — written imatrix.dat on disk — is the encoder's hint. For every linear projection in the network, it records a single number per input channel: how much that channel actually moved when the model was shown real text. The quantizer reads this hint and biases its grid toward the channels that mattered.

¶In practice the file is small — a few megabytes — even for a 70B model. It is metadata, not weights.

ii. A definition

For a linear layer y = W x, llama.cpp accumulates the second moment of the inputs over a calibration corpus of N tokens:

Ii  =  (1 / N)   Σt = 1..N   xi,t2 eq. 1

Here xi,t is the i-th input feature at token t, and the sum runs over every position in every calibration document. The resulting vector I, with one entry per input column of W, is the row of the imatrix that belongs to this tensor.

Mathematically this is the same object that turns up in the diagonal of the Fisher information for a quadratic loss: the expected squared activation is a proxy for how much does perturbing this column of W move the output. Doubling the precision of a column with twice the expected activation halves twice as much error.

¶Strictly, this is a heuristic — the true Fisher would also include downstream Jacobians. The squared-activation approximation works because the dominant contribution to per-weight loss is local, and because it can be gathered in a single forward pass with no backward graph.

iii. How it is gathered

The llama-imatrix binary reads a corpus of plain text, runs the model in forward-only mode at full precision, and at each linear layer captures the input tensor just before the matmul. Squared values are accumulated into a per-tensor float buffer. The corpus is typically a few hundred thousand tokens — a slice of wikitext, a domain mix, a curated chat log:

# gather statistics over a calibration corpus
llama-imatrix \
  -m models/gemma-2-9b.f16.gguf \
  -f calibration/wikitext.train.raw \
  -o gemma-2-9b.imatrix \
  --chunks 200 -c 512

# then quantize, passing the hint in
llama-quantize \
  --imatrix gemma-2-9b.imatrix \
  models/gemma-2-9b.f16.gguf \
  models/gemma-2-9b.IQ3_M.gguf \
  IQ3_M

The estimate stabilizes faster than one might expect. Most channels carry consistent statistics across natural text — a hundred thousand tokens is usually within a few percent of the limit, and going to ten million changes the resulting perplexity by less than the run-to-run noise.

¶Stability is a property of the second moment, not of the model. A long-tailed distribution would need orders of magnitude more samples; activations in well-trained transformers are not long-tailed.

iv. Where it enters

During quantization, the blockwise quantizers — Q4_K, Q5_K, the IQ family — solve a weighted least-squares problem per block of n weights:

minimize   Σi   Ii · ( Wi − Q(Wi) )2 eq. 2

instead of the unweighted form. The scale, the offset, and (for the IQ-quants) the codebook selection are all chosen to minimise this importance-weighted error. Columns with large Ii pull the grid toward themselves; columns with small Ii are allowed to drift.

The effect is largest for the most aggressive formats. At IQ2 there are essentially four representable magnitudes per block and every bit of redirected precision is consequential. At Q4_K_M the budget is generous enough that the imatrix moves perplexity by a few percent. At Q8_0 there is so much headroom that the file is — for most models — indistinguishable from one quantized without a hint.

v. What it buys

The improvement, measured by perplexity on held-out text, scales inversely with the bit budget. Rough magnitudes, averaged across a handful of dense decoder models:

IQ2_XS−12.4%
IQ3_M −7.8%
Q4_K_M−3.9%
Q5_K_M−1.6%
Q6_K −0.5%
Q8_0 ±0.1%

The improvement is larger still for architectures whose activations are heterogeneous: mixtures of experts, where many channels are silent on most tokens; models with attention sinks; models with long-tailed embedding statistics. For these, an unweighted quantizer is spending precision on routes that the network rarely uses. The imatrix redirects it.

vi. Failure modes

The calibration corpus matters more than any other choice. An imatrix gathered on English Wikipedia will under-weight channels that activate on source code, on non-Latin script, on chat formatting tokens, on tool-call syntax. The resulting quant will be quietly worse on those distributions — and the perplexity number on wikitext will not tell you. Most this quant is worse than the official one reports trace back to a mismatch between calibration text and target usage.

The remedy is unglamorous: gather calibration data that resembles what the model will be asked to do. A coding model deserves a code-heavy calibration mix. A multilingual model deserves multilingual text. The imatrix is a hint, and a hint shaped wrong points the encoder in the wrong direction.

¶A practical compromise that holds up: roughly sixty percent natural language across several languages, twenty-five percent code, fifteen percent structured chat. Resist the urge to over-tune; the second moment is robust, not delicate.
